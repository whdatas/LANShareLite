# 临时共享与自动清理：让分享目录用完不留垃圾

培训资料、展会安装包、测试日志和现场交付文件往往只有几小时或几天的价值。手动清理很容易忘，最终共享目录变成无人敢删的杂物堆。

[狼享 LAN Share Lite](https://lan-share.whdatas.com/lite-version)  的临时共享模式可以按文件最后修改时间自动清理过期内容。

## 开启临时模式(必须谨慎，避免误删重要文件)

在 `app.yaml` 中配置：

```yaml
temporary: true
temp_ttl: "24h"
preserve:
  - readme.txt
  - .gitkeep
  - docs/**
```

- `temporary`：是否启用后台清理；
- `temp_ttl`：超过多久未修改视为过期；
- `preserve`：永久保留的文件或路径模式。

修改后重启服务。

## 它适合哪些场景

### 培训与会议

会前上传讲义和安装包，会后 48 小时自动清理。保留 `readme.txt` 作为下载说明。

### 展会与现场服务

临时向同一 Wi-Fi 下的访客分发驱动、手册或演示文件，撤场后无需逐项整理。

### 开发测试

测试人员上传日志和构建包，保留时间设置为 72 小时，避免临时产物长期占用磁盘。

## 清理机制的边界

自动清理依据时间和规则，不理解业务价值。配置前必须：

1. 使用独立临时目录，不要直接指向重要资料库；
2. 先在测试文件上验证 TTL；
3. 为必须保留的内容配置 `preserve`；
4. 对重要数据另做备份；
5. 检查系统时间和时区是否正确。

不要把自动清理当作备份或回收站。被清理的文件未必可以恢复。

## 推荐目录设计

```text
临时共享/
├── readme.txt        # preserve
├── incoming/         # 允许上传
├── downloads/        # 临时分发
└── docs/             # preserve: docs/**
```

## 从临时共享到正式协作

Lite版本 的 TTL 适合按时间清理。如果需求变成“分享链接单独设置有效期”“只有指定用户能访问”“删除进入回收站”“管理员可以审计”，就不应继续依靠一个公共临时目录，应升级到 [Pro](https://lan-share.whdatas.com/) 的分享、身份和治理能力。
