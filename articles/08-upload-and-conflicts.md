# 拖拽上传、目录上传与冲突处理：局域网传文件实战

真正影响文件共享体验的，往往不是“有没有上传按钮”，而是一次要传几十个文件、整个目录，或者目标位置已经存在同名文件时会发生什么。

[LAN Share Lite](https://lan-share.whdatas.com/lite-version) 把这些操作放在浏览器中完成，访问端不需要安装客户端。

## 上传单个或多个文件

打开共享页面后，可以点击上传按钮选择文件，也可以把文件直接拖入上传区域。浏览器会显示：

- 当前文件名；
- 单文件上传进度；
- 已完成数量和总数量；
- 成功或失败提示。

建议第一次先上传一个小文件，确认共享目录和磁盘权限正确，再处理正式资料。

## 上传整个目录

点击“上传目录”后选择文件夹，浏览器会递归读取其中的文件，并在服务端重建相对目录结构。

它适合：

- 网站静态资源；
- 摄影或设计项目；
- 软件安装包及说明文件；
- 分层整理的培训资料。

需要注意：浏览器上传目录依赖 `webkitdirectory` 能力，现代桌面浏览器支持较好，部分手机浏览器可能只允许选择文件。

## 同名文件如何处理

直接覆盖同名文件很危险，尤其是多人把文件上传到同一目录时。Lite 会先调用冲突检查接口，列出已经存在的目标文件，再由用户确认是否继续。

操作前建议：

1. 检查当前所在目录；
2. 查看冲突文件列表；
3. 不确定时先改名或新建子目录；
4. 重要文件保留本地备份。

Lite 的冲突确认不是完整版本管理。需要保留历史版本、按用户追踪覆盖行为时，应考虑 Pro。

## 新建目录与整理

网页可以新建目录，并通过面包屑在层级间移动。一个实用的目录约定是：

```text
共享根目录/
├── 00-说明与规范/
├── 10-待处理/
├── 20-已确认/
└── 90-临时文件/
```

目录规则比把所有文件扔在根目录更有效。Lite 支持搜索，但搜索不能代替基本的信息结构。

## 上传失败怎么排查

| 提示或现象 | 检查项 |
|---|---|
| 文件超过限制 | 调整 `max_upload_size` |
| 上传开始后立即失败 | 共享目录写权限、磁盘空间 |
| 传到一半中断 | Wi-Fi、`write_timeout`、浏览器休眠 |
| 目录结构不完整 | 浏览器目录选择能力、特殊文件名 |
| 403 或功能禁用 | 检查 `enable_upload` |

配置示例：

```yaml
enable_upload: true
enable_delete: false
max_upload_size: "20GB"
write_timeout: 7200
```

## 删除功能为什么默认应谨慎

Lite 没有账号体系。只要客户端能访问页面，启用删除后就可能执行删除。因此资料分发或多人提交场景建议：允许上传，关闭删除。

当团队需要“只有管理员能删除”“某部门只能写自己的目录”“删除必须进入回收站并记录操作者”时，就已经进入 Pro 的权限治理范围。
