# 五个真实场景：家庭、工作室、设计交付、研发内网与现场服务

一款工具是否值得用，关键不是功能列表，而是它能否融入具体工作。下面五个场景展示 [狼享 LAN Share Lite](https://lan-share.whdatas.com/lite-version) 适合做什么，也说明什么时候应升级。

## 场景一：家庭电脑与手机互传

需求：把电脑中的电影传到平板，把手机照片集中到电脑。

做法：在家庭电脑安装 Lite，手机扫描二维码访问。关闭删除，按需开启上传。

为什么适合：设备系统混杂、没有账号管理需求、网络可信。

## 场景二：小型设计或摄影工作室

需求：集中交付大体积源文件、样片和导出视频。

做法：服务端接千兆有线网络，项目按目录组织，使用大文件配置和搜索。

升级信号：客户或员工需要不同访问范围，需要知道谁下载了素材，此时应评估 Pro。

## 场景三：研发内网分发构建产物

需求：开发向测试分发 APK、固件、日志和离线依赖包。

做法：使用 Linux 或 Docker 常驻，建立日期和版本命名规范，关闭删除，使用搜索定位历史构建。

升级信号：构建产物涉及项目隔离、下载审计和 LDAP 登录。

## 场景四：培训、会议和展会

需求：几十人临时下载讲义、驱动和安装包。

做法：准备独立 Wi-Fi，页面展示二维码，开启临时共享和 24/48 小时 TTL。

注意：访客网络常开启客户端隔离，部署前必须用另一台设备测试。

## 场景五：现场工程师与设备维护

需求：在没有稳定公网的工厂、机房或施工现场分发手册、日志和升级包。

做法：在工程师笔记本或小主机运行 Lite，通过便携路由器组成局域网。

升级信号：多个现场节点需要统一状态、告警和全局搜索时，应评估 Enterprise。

## 场景选择表

| 场景特征 | Lite | Pro/Enterprise |
|---|---|---|
| 可信网络、无需账号 | 很适合 | 通常没必要 |
| 临时分发、扫码下载 | 很适合 | 视审计要求而定 |
| 不同人员访问不同目录 | 不适合 | 适合 |
| 需要操作追踪 | 不适合 | 适合 |
| LDAP/AD 登录 | 不支持 | 适合 |
| 多节点统一管理 | 单实例为主 | Enterprise 更适合 |

先用 Lite 验证共享方式是否适合团队，是一种低成本试点；当问题从“传输效率”变为“权限和责任”时再升级，通常比一开始建设复杂平台更稳妥。
