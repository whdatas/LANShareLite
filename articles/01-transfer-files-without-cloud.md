# 不用微信和 U 盘，怎样在局域网内快速传大文件

给同事传一个 8GB 的视频、虚拟机镜像或设计源文件，常见做法通常都不够理想：微信有大小和保存体验限制，网盘要先上传公网再下载，U 盘需要来回走动，还可能遇到格式、容量和病毒问题。

如果两台设备本来就在同一个办公室或家庭网络里，更直接的办法是：让文件只在局域网中传输。

## 先说结论

LAN Share Lite 可以把一台电脑上的目录变成浏览器可访问的文件站。服务端安装一次，其他 Windows、macOS、Linux、iPhone、iPad 或安卓设备不需要安装客户端，打开浏览器即可上传和下载。

它适合这些情况：

- 文件较大，不想绕行公网；
- 设备系统不同，不想研究 SMB 兼容问题；
- 对方只需要浏览器，不方便安装软件；
- 文件只在可信局域网内流转；
- 想把一台旧电脑临时变成文件中转站。

## 三步完成第一次传输

### 第一步：在一台电脑上安装 Lite

从 [狼享 LAN Share 官网](https://lan-share.whdatas.com/lite-version) 下载对应系统的 Lite 安装包。Windows、macOS 和 Linux 可以安装为后台服务，也可以使用 Docker。

安装后默认使用 `8080` 端口，共享文件存放在系统数据目录。以 Windows 为例：

```text
C:\ProgramData\lan-share-lite\user_data
```

### 第二步：找到访问地址

在服务端浏览器打开：

```text
http://localhost:8080
```

页面会显示局域网 IP、二维码和可访问 URL，例如：

```text
http://192.168.1.100:8080
```

支持 mDNS 的设备还可以尝试：

```text
http://lan-share-lite.local:8080
```

### 第三步：在另一台设备上传或下载

确保两台设备连接同一个路由器或同一办公网络，然后在另一台设备输入上面的 IP 地址。电脑可以直接拖拽文件或整个目录，手机可以扫描二维码。

传输过程不会先上传到 LAN Share 的第三方服务器，速度主要取决于你的有线网络、Wi-Fi 和磁盘性能。

## 为什么它适合大文件

LAN Share Lite 默认允许最大 `10GB` 上传，可在 `app.yaml` 中调整：

```yaml
max_upload_size: "50GB"
write_timeout: 7200
read_timeout: 7200
```

下载基于标准 HTTP Range，浏览器和下载工具可以进行分段读取；页面还提供上传进度和连接质量检测。对于持续传输大文件，千兆有线网络通常比跨房间 Wi-Fi 稳定得多。

## 使用前必须知道的边界

Lite 的定位是可信局域网内的轻量共享，不包含账号登录、部门权限和操作审计。因此：

- 不要直接暴露到公网；
- 不要在陌生公共 Wi-Fi 上开放；
- 敏感资料建议开启 HTTPS；
- 不需要删除时关闭删除功能；
- 多人长期使用且需要追责时，应升级到 Pro。

## 它和 Pro 的区别是什么

Lite 解决的是“怎样快速传文件”。当团队开始问“谁能看这个目录”“谁删除了文件”“能否使用公司域账号登录”时，问题已经变成企业文件治理。这时 Pro 提供的用户、角色、ACL、审计和 LDAP 才是核心价值。

如果你现在只是想摆脱微信、网盘和 U 盘，可以先从免费的 LAN Share Lite 开始。
