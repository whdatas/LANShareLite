# Windows、macOS、Linux 和手机之间传文件，有哪些靠谱方法

跨平台传文件难的地方往往不是“没有工具”，而是每种工具都带着自己的前提：系统共享要配置账号和协议，点对点工具要求双方安装客户端，网盘依赖公网，U 盘又需要物理接触。

没有一种方法适合所有场景，先看需求再选工具更实际。

## 常见方案对比

| 方案 | 优点 | 局限 | 更适合 |
|---|---|---|---|
| U 盘/移动硬盘 | 简单、离线 | 需要来回插拔，有丢失和病毒风险 | 一次性离线迁移 |
| 微信/企业 IM | 人人会用 | 大文件、压缩、过期和公网传输 | 小文件临时发送 |
| 公有云盘 | 远程访问方便 | 依赖账号、外网、配额和上传速度 | 跨地域长期同步 |
| SMB 系统共享 | 原生、适合挂载盘符 | 跨系统权限和发现配置较复杂 | 固定办公网、熟悉 IT 管理 |
| FTP/SFTP | 标准成熟、工具多 | 普通用户操作门槛高 | 技术人员和自动化任务 |
| 点对点工具 | 发送体验直接 | 通常要求双方安装并在线 | 两台设备即时互传 |
| LAN Share Lite | 浏览器访问、跨平台、文件留在本地 | Lite 无账号和细粒度权限 | 可信局域网内多人浏览和传输 |

## 什么时候浏览器共享最省事

下面几个条件同时出现时，狼享 LAN Share Lite(https://lan-share.whdatas.com/lite-version) 这类方案通常更合适：

- 服务端可以长期在线；
- 访问者设备很多、系统混杂；
- 不希望每个访问者安装 App；
- 文件应保存在指定电脑或服务器；
- 访问范围主要在同一局域网；
- 需要的不只是“发送一次”，还包括浏览和搜索。

例如，摄影工作室把当天样片放到一个目录，所有同事和客户现场通过浏览器挑选；这比逐个点对点发送更自然。

## 什么时候不该选 Lite

如果双方不在同一网络，又没有 VPN 或安全的反向代理，公有云盘可能更省事。如果文件必须自动双向同步，专业同步工具更合适。如果企业要求每一次访问都有身份和审计，应该使用 LAN Share Pro 或其他企业文件平台。

## 跨平台使用示例

假设 Windows 电脑运行 Lite版，IP 是 `192.168.1.20`：

```text
Windows 服务端：http://localhost:8080 或者 http://192.168.1.20:8080
Mac 浏览器：http://192.168.1.20:8080
iPhone/安卓：扫描页面二维码 或者 http://192.168.1.20:8080
Linux：浏览器 http://192.168.1.20:8080 或 curl/wget 访问下载链接
```

如果 mDNS 在当前网络中可用，也可以尝试：

```text
http://lan-share-lite.local:8080
```

## 选型的一个简单判断

- 两台设备临时互传：优先点对点工具；
- 同一团队反复访问一个共享目录：优先 Lite；
- 多部门权限、审计和统一账号：优先 Pro；
- 跨地域、随时外网访问：评估 VPN、对象存储或企业云盘。

工具没有绝对高下，关键是不要用“临时传文件”的方案承担“长期企业治理”的责任。
