# 免费版什么时候够用，什么时候应该升级 Pro

软件选型中最容易犯的两个错误，一是为了“以后可能需要”一开始就部署复杂系统，二是团队已经出现明显管理风险，却仍把临时工具当正式平台。

[LAN Share Lite](https://lan-share.whdatas.com/lite-version) 和 [LAN Share Pro](https://lan-share.whdatas.com) 的分界，不在文件大小，也不简单取决于人数，而在于是否需要身份和治理。

## 这些情况 Lite版本 通常已经够用

- 家庭成员在可信网络中互传照片和视频；
- 个人在电脑、手机和平板之间交换文件；
- 工作室临时分发设计稿、安装包和素材；
- 培训、展会或现场服务短期共享资料；
- 小团队所有成员可以访问同一目录；
- 不需要知道具体是谁下载或删除了文件；
- 可以通过网络和防火墙控制访问范围。

在这些场景里，Lite 简单、免费、无需账号反而是优势。不要为了显得“企业级”增加不必要的登录和审批。

## 出现第一个升级信号：需要区分“谁”

当你开始提出这些问题：

- 新员工和外包人员能看到哪些文件？
- 财务目录能否只给财务人员？
- 离职员工权限如何立即回收？
- 管理员和普通用户能否拥有不同操作？

这说明网络边界已经不够，需要用户、动态角色和 ACL。Lite 的公共共享模型不适合继续承担这个责任。

## 第二个信号：需要回答“发生了什么”

如果团队需要知道：

- 谁上传、下载或删除了文件；
- 某次误操作发生在什么时间；
- 是否能导出或验证审计记录；
- 安全事件后如何追溯；

就需要审计能力。没有身份，就无法建立可靠的操作者记录。

## 第三个信号：账号不能单独维护

企业已经使用 LDAP 或 Active Directory 时，再维护一套孤立账号会增加入离职管理成本。需要统一登录、组到角色映射和定期同步时，应评估 [Pro/Enterprise版本](https://lan-share.whdatas.com) 的 LDAP 能力。

## 第四个信号：运维范围扩大

- 磁盘快满时需要告警；
- 登录失败激增需要通知；
- 多台服务器要统一查看；
- 多个节点之间需要发现、同步或全局搜索；
- 需要升级保障、交付支持和明确责任人。

这些已经不是单机工具的典型边界，更接近 [Enterprise](https://lan-share.whdatas.com) 运维。

## 一个实用评分表

每项“是”记 1 分：

- [ ] 有 10 人以上长期使用；
- [ ] 存在部门或项目隔离；
- [ ] 文件包含客户、财务或源码等敏感资料；
- [ ] 需要账号登录；
- [ ] 需要操作审计；
- [ ] 使用 LDAP/AD；
- [ ] 需要告警通知；
- [ ] 有多个地点或节点；
- [ ] 需要正式技术支持。

参考判断：

- `0-2`：Lite 通常足够；
- `3-5`：建议试用 Pro 并验证权限模型；
- `6+`：建议按 Enterprise 项目规划，不要只比较许可证价格。

这不是硬性规则。一个只有 3 人的财务团队也可能强烈需要审计，而一个 20 人的公开素材工作室仍可能只用 Lite。

## 正确的升级方式

先用 Lite 验证目录结构、网络和用户习惯，再盘点角色、ACL 和审计要求。升级前回答三个问题：

1. 哪些目录需要隔离？
2. 哪些操作必须留痕？
3. 账号从哪里来、由谁维护？

如果这三个问题没有答案，换成 Pro 也不会自动获得良好治理。
